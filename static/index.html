⚠️ Schema-Aware Prototype

The DAIL export contains Case_Table and Docket_Table as schema metadata (column definitions), not case/docket records. Only Document_Table (863 rows) and Secondary_Source_Coverage_Table (389 rows) contain actual data. Cases shown below are stub records synthesised from Case_Number FK references — ready to be enriched when real case data becomes available.

🔄 Full Reload

Loads all RAW tables from Excel, then runs the full transform + validate pipeline. Takes an automatic curated-layer snapshot on success.

Δ Incremental Delta Load NEW

Computes a per-row SHA-256 checksum for every Excel row and only writes rows that are new or changed. Every processed row produces a RawDeltaLog entry (insert / update / skip) for complete provenance. Transforms the curated layer only when at least one raw row changed, then auto-snapshots.

📈 Data Versioning & Snapshots

Every pipeline run automatically freezes a point-in-time copy of the curated layer. Use Diff to see exactly what changed between any two snapshots.

Provenance Enforcement Rules

  • Every PATCH / POST to curation endpoints MUST include editor_id + reason
  • At least one of citation_id or citation_justification must be provided (422 otherwise)
  • Case-name changes are additionally tracked in case_caption_history
  • The change_log table is append-only — no deletes allowed
  • All write endpoints require X-API-Key header
